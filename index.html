Git

  • The git cheat sheet is a good reference for git commands that can be used in the command line (GitBash). The Git workflow video is a useful reference for understand how to upload and download changes to and from Git.
  • I should work on repositories cloned from GitHub locally on a new feature branch. I should always pull the most recent main branch from Git before committing any changes from my feature branch.
  • git status: tells you which github branch you're in.
  • git checkout -b branch-name: creates a new branch.
  • git add -A: adds a new branch to a working branch.
  • git commmit 'm ("tailor your message here"): adds a message that describes what the commit includes.
  • git pull origin main: checks local branch is in sync with the base branch in GitHub.
  • git push origin branch-name: pushes changes to the remote branch.
